# Application Bullets — 자소서 삽입용 문장 세트

각 문항에 자연스럽게 삽입 가능한 짧은 문장들입니다. 그대로 쓰지 말고 자신의 맥락과 문장 흐름에 맞춰 변형해서 사용하세요.

---

## 문항 1 (입사 희망 이유 + 목표)

- "규정을 쿼리로, 데이터를 보고로, AI를 검토 보조로 연결하는 일을 업(業)으로 삼기 위해 준비해왔습니다."
- "머신러닝 기반 STR 위험도 측정과 생성형 AI를 활용한 규제 모니터링이 이미 실무에 적용되고 있다는 점은, 제가 준비해온 법규·데이터·AI의 3축이 학습 단계를 넘어 운영 환경에서 검증될 수 있는 드문 조건이라고 판단했습니다."
- "인턴십 기간의 AI 과제에서는 'regulatory news 요약 에이전트' 또는 'alert 검토 초안 자동화'를 human-in-the-loop 구조로 설계하는 것을 목표로 하고 있습니다."
- "법학 전공자가 사후 분쟁의 대리인이 아니라 사전 예방의 설계자로 일할 수 있는 영역이 디지털 내부통제라고 판단했고, 이를 위해 규정과 데이터·AI를 연결하는 학습을 직접 손으로 진행해왔습니다."

## 문항 2 (관심 계기 + 본인 강점)

- "국세청 제13차 국세실무수습 과정에서 자금의 경제적 실질을 따라가는 분석 방식을 접하며, 규정이 단순히 해석의 대상이 아니라 데이터 위에서 작동해야 작동한다는 점을 처음 체감했습니다."
- "수험생 300여 명의 성적·학사 데이터를 8개월간 전담하며 '요청 접수 → 반영 → 최종 크로스체크'의 3단계 검증 체크리스트를 자체 설계해 운영 오류 0건을 기록했습니다. 이는 AML 실무의 STR·CTR 보고에서 요구되는 무결성 감각과 같은 근육입니다."
- "법학 전공자로서 규정을 데이터 탐지 룰로 번역하는 연습을 위해 가상 거래 데이터 기반 AML 탐지 데모(github.com/diplomatieee/aml-monitoring-lab)를 직접 구현했습니다. 특금법·FIU 업무규정상 의심거래 유형 5개를 SQL 룰로 옮기고, 각 룰의 법적 근거를 별도 문서로 매핑하는 과정에서 '규정의 데이터화'가 실제로 어떤 정밀도를 요구하는지 체감했습니다."

## 문항 3 (당근 앱)

이 문항은 레포와 직접 연결이 낮습니다. 별도 뼈대를 사용하세요. 다만 마지막 연결 문장에 다음 톤을 참고:

- "플랫폼이 신뢰를 직접 만들어낼 수 없는 환경에서 작은 검증을 여러 층으로 쌓는 구조는, 비대면 채널에서 리스크를 통제해야 하는 컴플라이언스의 과제와 구조적으로 닮아 있습니다."

## 문항 4 (금융상품 AML 위험 + 방지안)

- "파킹통장의 고금리·자유입출금 특성은 Layering 단계의 허브로 활용될 위험, KYC 신고 프로필 대비 과다 거래 위험, 신규 계좌 단기 고액거래 위험을 동시에 내포합니다."
- "현행 제도로는 CTR, STR, EDD, 거래 모니터링 시스템의 rule-based + ML 혼합 탐지가 운영되고 있습니다."
- "담당자 관점에서 제안한다면 ① KYC 재확인 트리거에 상품 특성 반영 ② 파킹통장 전용 rule set 운영 ③ alert 1차 요약을 생성형 AI에 맡기되 최종 STR 제출 판단은 담당자에게 귀속 — 이 3단 구조입니다."
- "본 제안의 탐지 로직 일부는 학습 목적으로 가상 데이터 기반 데모(github.com/diplomatieee/aml-monitoring-lab)로 직접 구현해본 바 있습니다. 신규 계좌 30일 이내 5백만원 초과 거래, 동일 device 다계정 사용, 저소득 신고 고객의 과다 거래 등 파킹통장 리스크와 직접 닿는 룰을 SQL로 옮겨 보았습니다."
